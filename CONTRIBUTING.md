# Contributing

## 修改规则的建议流程

1. 先明确修改归属文档。
2. 使用 PR 提交修改。
3. 在 PR 中说明影响范围、适用风格和 QA 风险。
4. 如果是新增或修正游戏风格，请更新 `docs/02-style-knowledge.md`。
5. 如果是 Maker、预算、尺寸或归因问题，请更新 `docs/03-production-constraints.md`。
6. 如果是页面视觉主次、背景强度或组件层级问题，请更新 `docs/04-visual-hierarchy-brief.md`。
7. 如果是背景、程序化面板、固定角饰或状态产物规则，请更新 `docs/05-material-generation.md`。
8. 如果是由 QA 失败触发的修改，请先按 `docs/00-index.md` 归因，再回修对应文件。
